# Code of Conduct

Kakashi follows a contributor-friendly standard for open-source collaboration.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent while still being precise about technical issues.
- Keep discussions focused on the project, its users, and maintainable solutions.
- Give clear reproduction steps, evidence, and tradeoffs when reporting problems.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private credentials, tokens, logs, or repository contents.
- Repeatedly derailing issues or pull requests away from the project topic.
- Using the project spaces to pressure maintainers for private support.

## Reporting

For conduct concerns, open a minimal public issue only if it contains no private information, or use the security reporting path in `SECURITY.md` when the issue involves credentials, private data, or abuse risk.

Maintainers may edit, hide, lock, or remove content that violates this code of conduct, and may limit participation for repeated or severe violations.
