# Support

Use the public GitHub issue templates for reproducible bugs, feature requests, and documentation problems.

Before opening an issue:

- Run `kakashi doctor`.
- Check the current README and release notes.
- Include the Kakashi version, operating system, Node.js version, and the exact command you ran.
- Redact API keys, GitHub tokens, Codex access tokens, private repository names, private logs, and generated project secrets.

For security-sensitive reports, do not open a public issue with exploit details or secrets. Follow `SECURITY.md`.

Kakashi is an open-source project. Public issues are the default support channel; private operational support is not provided through the repository.
